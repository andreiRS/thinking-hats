# 🎩 The panel — run all six hats

Take the idea on the table through the full sequence of hats, one at a time. Lens
definitions for every hat are in `../../hats/lens.md` — follow each one's focus.

If no idea is given, ask what to run the panel over first.

## Sequence

Go in this order (de Bono's typical flow; adjust if the user asks):

1. ⚪ **White** — the facts and what's missing
2. 🔴 **Red** — gut reaction
3. ⚫ **Black** — risks and failure modes
4. 🟡 **Yellow** — value and best case
5. 🟢 **Green** — alternatives and new options
6. 🔵 **Blue** — synthesis (below)

## Compressed output per hat

To avoid a wall of text, each hat is compressed: **the lens (A) + takeaways (C) only** —
**skip the per-hat questions.** Open questions are collected by the blue hat at the end.
Give each hat its own short block with the colored emoji heading. Red stays gut-only.

## Blue-hat close

End with a 🔵 blue-hat block that:
- **Open questions** — gather the questions the hats raised that still need answers.
- **Synthesis** — where the balance lands across the hats and the suggested next move.

Keep each block tight. The value is seeing all the angles together, not exhausting each one.
